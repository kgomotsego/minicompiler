BRAZE MINI COMPILER

World of Modern compilers

"This is product of the Braze company developed M.K. Africa and K.F. Josephs.

MJ Blue was also involved at a later stage of the development.

The language dominating in the mini compiler is pascal.

Basic assembly x86 was included in some of the procedures.

The mini compiler was inspired by the great Julian Pareshki

from France who built the first open-Source compiler.

The Braze Mini Compiler is also open-source"

Figure 1

The procedure GetChar allows the program to read new character from the input stream.

The procedure Error will check the data type in its argument, if the data type

entered is not a string then the program will display an error. The Abort procedure

stops the current processing once the error is found.

Figure 2

The procedure Expected reports what was expected. The Match procedure will match the

specific input character to the program. The function IsAlpha will allow the program

to recognize uppercase characters. The function IsDigit will allow the program to recognize digits.

Figure 3

In the function GetName ,the condition statement ” if not IsDigit(Look) then Expected

(‘Name’) “ tells the user to enter a digit and

if not then the program will tell

the user that a name is expected. In the function GetName, the condition statement “if

not IsDigit(Look) then Expected (‘Integer’) tells the user to enter a digit and if

not then the program will tell the user that an integer is expected. The procedure

Emit produces an output string without moving the cursor to the next line. The

procedure EmitLn will produce an output string and move the cursor to the next line.

Figure 4

In the procedure Term, the functions Match ( ‘(‘ ) and Match ( ‘)‘ ) have been added to

enable the program to recognize parentheses in an expression. In the procedure Add,

procedure Match is called to look for “+” in any expression that may be input. Procedure

Term then moves the first digit to D0 and the procedure expression moves contents of

register D0 to D1. The Add procedure will then add the contents of registers D1 and D0

where the D1 contains the second number to be added. In the procedure Subtract,

procedure Match is called to look for “-” in any expression that may be input.

Procedure Term then moves the first digit to D0 and the procedure expression moves

contents of register D0 to D1. The Subtract procedure will then add the contents of

registers D1 and D0 where the D1 contains the second number to be subtracted.

Figure 5

In the procedure Divide, procedure Match is called to look for “/” in any expression

that may be input. Procedure Term then moves the first digit to D0 and the procedure

expression moves contents of register D0 to D1. The following digit after the division

operator is then moved to D1 and the procedure Divide finally divides the contents of

register D1 by the contents of register D0. In the procedure Multiply, procedure

is called to look for “*” in any expression that may be input. The procedure Term will

move the first recognized digit to register D0, the Multiply procedure will then multiply

the contents of registers D1 and D0 where the D1 contains the second number to be

multiplied. The procedure Expression can recognize math operators and call necessary

procedures (line 176-179) for recognition and translation of general expressions. The

procedure Expression contains a loop structure (line 173) which allows the procedure to

check for more than one math operator. The procedure only recognizes +, -, * and /

(line 173) since their procedures are defined as shown in figures 4 and 5. The procedure

has a line of code that that will send an error message when no math operator is

not found (Line 180).
